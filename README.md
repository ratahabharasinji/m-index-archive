# m-index-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态形式存储，可被直接访问或用作索引。

## 项目简介

`m-index-archive` 是一个通用型 HTML 页面归档仓库。其中存放的页面内容彼此独立，不针对特定域名或网站。仓库的主要目的是提供一个结构化的存储位置，方便对不同来源或用途的 HTML 文件进行集中管理和版本记录。

## 目录说明

```
.
├── pages/        # 存放独立 HTML 页面文件
├── assets/       # 页面引用的静态资源（如 CSS、JS、图片等）
└── README.md     # 本说明文件
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名通常反映其内容或用途。
- `assets/`：若页面需要额外的样式、脚本或图片，统一放置在此目录下，便于维护。

## 页面归档说明

- 本仓库中的 HTML 页面均为静态文件，不依赖后端服务。
- 页面内容可包含文本、链接、多媒体等元素，但均以独立文件形式存在。
- 归档的目的在于保留页面原始结构与信息，不保证所有页面长期可用或更新。
- 部分页面可能引用外部资源（如 CDN 字体、库文件），这些资源的可用性由原提供方决定。

## 维护说明

- 欢迎通过 Issue 或 Pull Request 提交新的归档页面，或对已有页面提出改进建议。
- 新增页面请放入 `pages/` 目录，并尽量保持文件命名简洁清晰。
- 若引用了外部资源，建议在页面注释中注明来源，以便后续追踪。
- 仓库维护者会不定期清理无效引用或过于陈旧的内容，但不保证实时更新。
